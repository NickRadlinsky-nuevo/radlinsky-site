# Прототип статьи блога/журнала для radlinsky.com.ua

Единый формат оформления статей (клинические случаи, материалы из журнала «ДентАрт»).
Цель: все статьи выглядят как одна серия, страница лёгкая и быстрая.

**Эталоны в репозитории:**
- `white-spots-removal-and-volume-restoration.html` — короткая статья с **пошаговым протоколом** (нумерованные шаги)
- `statti-ponomarenko-restavratsiya-zubiv-i-zubnykh-riadiv.html` — большой **клинический случай из журнала** (аннотация + разделы + сетки фото)

Оба используют один визуальный язык — берите ближайший по типу как основу.

---

## 1. Структура страницы (сверху вниз)

| Блок | Класс | Обязателен |
|---|---|---|
| Шапка сайта | `header.site` | ✅ (как на всех страницах — лого, меню, соцсети, бургер) |
| Хлебные крошки | `.crumb` | ✅ `Аполлонія / Статті / <назва>` |
| Титул | `.art-head` | ✅ кикер (рубрика/источник) → `h1` → `.subt` подзаголовок → `.en-title` англ. титул (если есть) |
| Автор | `.author` | ✅ круглый портрет + имя + аффилиация. **Без личного email/телефона автора** |
| Аннотация | `.abstract` | для журнальных статей: Резюме / Ключові слова / Abstract / Key words |
| Тело | `.art-body` | абзацы `p`, лид `p.lead`, диагноз `p.diag`, стих/цитата `p.verse` |
| Заголовки разделов | `h2` (раздел), `h3` (подраздел) | |
| Фото | `.ph-grid` (2 кол.) или `.ph-fig` (grid + `figcaption`) | |
| Протокол (если пошаговый) | `ol.protocol` (нумерованные кружки) | как в white-spots |
| Висновки | `.concl` (тёмный petrol-блок) | ✅ |
| Література | `.refs` (нумерованный список) | если есть в источнике |
| CTA | `.cta` | ✅ ссылка на курс/расписание |
| Подвал | `footer.site` | ✅ как на всех страницах |

## 2. Стиль (тот же, что у основного сайта — НЕ брендбук Аполлонии)

CSS зашит в `<style>` каждой статьи, менять не нужно. Токены:
- Цвета: `--enamel #F7F4EC` (фон), `--porcelain #FCFBF6` (карточки), `--ink #1B1916`, `--ink-soft #56514A`, `--petrol #133B38` (тёмные блоки), `--brass #B0894F` (акцент)
- Шрифты: `Spectral` (заголовки, лид, цитаты) + `Manrope` (текст) — грузятся неблокирующе (preload + onload)
- Ширина контента статьи: `max-width:820px` по центру; фото-сетки — на всю ширину `.wrap` (1180px)
- Радиусы 6–12px, тонкие линии `--line`

## 3. Работа с изображениями (главное для скорости)

**Извлечение из PDF:**
```bash
pdfimages -f <first> -l <last> -j -p "<file.pdf>" out   # -j = сохранить JPEG как есть
pdftotext -enc UTF-8 -f <first> -l <last> "<file.pdf>" text.txt
```
⚠️ Первая/последняя PDF-страница ≠ страница журнала (обычно смещение +2 из-за обложки/анонсов).

**⚠️ Ориентация — обязательная проверка!**
`pdfimages` часто отдаёт фото **перевёрнутыми на 180°**. Порядок действий:
1. Развернуть все извлечённые фото: PIL `im.transpose(Image.ROTATE_180)`
2. **Сверить с исходником** (не только верх/низ, но и зеркальность L/R!): отрендерить страницу-эталон `pdftoppm -f N -l N -r 90 -png` и сравнить асимметричные снимки (окклюзионные виды с языком/нёбом — самые показательные). Если после ROTATE_180 фото зеркальны — значит нужен был `FLIP_TOP_BOTTOM`, не поворот.

**Оптимизация (цель ~10–15 КБ на фото):**
```python
im.convert("RGB").save(name, quality=82, optimize=True, progressive=True)
```
- Прогрессивный JPEG, качество 82
- В тегах `<img>`: `width`/`height` (нет скачков вёрстки, CLS=0), `loading="lazy"` на всех, кроме первого-двух, `decoding="async"`
- Итог эталонной статьи: 53 фото = ~740–900 КБ, первый экран открывается мгновенно

## 4. Правила контента

- **Текст источника не менять** — переносить дословно (это материалы своего журнала «ДентАрт»). Разбивать на абзацы/разделы по смыслу.
- **Убирать личные контакты автора** (email, личный телефон) из вёрстки и из Schema.
- Подписи к фото (`figcaption`) сохранять из оригинала.
- Диагноз выделять `p.diag`, лид-абзац — `p.lead`.
- Длинные протоколы прятать в `ol.protocol` или (для очень длинных списков) в `<details>`.

## 5. SEO и трекинг (копировать из эталона)

- `<title>`, `meta description`, `canonical` (чистый slug на `radlinsky.com.ua`)
- Open Graph + Twitter (`og:image` — первый показательный клинический снимок)
- JSON-LD `Article`/`ScholarlyArticle`: author (без email), publisher (НЦ Аполлонія + лого), `isPartOf` Periodical «ДентАрт» ISSN `1993-2170`, `inLanguage:"uk"`
- GA4 `G-7E639R6FKT` + Meta Pixel `1357557515478741` (тот же блок, что на сайте)
- slug: `<тема-латиницею>` (напр. `white-spots-removal-and-volume-restoration`) — если статья была на старом сайте, брать её старый URL 1:1

## 6. Чеклист публикации

1. HTML в корне репо `<slug>.html`, фото в `images/` (уникальный префикс, напр. `pon-NN.jpg`, `white-spots-N.jpg`)
2. Проверить: `grep -o '{{' <slug>.html` пусто; фото ориентированы верно (сверка с PDF); email автора убран
3. Локальный просмотр (`open <slug>.html`)
4. Решить перелинковку: ссылка со страницы «Статті»/раздела блога (сейчас статьи не залинкованы — добавлять осознанно)
5. `git add <slug>.html images/… && git commit && git push` → Cloudflare Pages (~1–2 мин)
6. Проверить live: страница 200, фото 200, OG-превью в мессенджере

## 7. Как заказать статью у Claude

Прислать PDF (или ссылку) + указать: какую статью/страницы, автор и его портрет (если новый), нужный slug. Claude извлечёт текст и фото, проверит ориентацию, оптимизирует, соберёт по этому прототипу и покажет локально до деплоя.
